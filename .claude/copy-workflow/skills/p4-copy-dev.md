# Steps 13–17 | Framework application, first draft, variants, persona test, India alignment

## Phase Goal

## Step 13 — Framework Application

### Agent actions
Load the approved framework from Phase 1 Step 4. Apply the structural template to the
positioning statement from Phase 3 Step 12. Do not write full copy yet — map the structure.

**Framework templates:**

### AIDA (Landing page hero, awareness stage)
```
A — ATTENTION: [headline — single most important claim, ≤8 words]
I — INTEREST: [subhead — proof-backed expansion of the headline, ≤20 words]
D — DESIRE: [3 bullet props — each proof-backed, specific, persona-resonant]
A — ACTION: [CTA — action verb + specific outcome]
```

### PAS (Cold email, outreach, awareness→consideration)
```
P — PROBLEM: [name the specific pain — their words, not marketing language]
A — AGITATE: [amplify the consequence — what happens if this pain goes unsolved]
S — SOLUTION: [position SAT as the specific resolution — proof-backed]
[CTA]
```

### BAB (Feature announcement, consideration stage)
```
B — BEFORE: [describe current state without SAT — specific, visceral]
A — AFTER: [describe the state with SAT — specific, proof-backed]
B — BRIDGE: [SAT is the mechanism that moves them from before to after]
[CTA]
```

### PASTOR (Case study, consideration→decision)
```
P — PROBLEM: [the challenge the firm faced — specific]
A — AMPLIFY: [what was at stake if unsolved]
S — STORY: [the turning point — where SAT entered the workflow]
T — TRANSFORMATION: [before/after — specific metrics]
O — OFFER: [what SAT offers to replicate this result]
R — RESPONSE: [CTA]
```

### Hook → Insight → Proof → CTA (LinkedIn, awareness)
```
HOOK: [provocative single line — question, data point, or contrarian claim]
INSIGHT: [2–3 sentences expanding the hook with specific knowledge]
PROOF: [one concrete data point, process comparison, or named example]
CTA: [one line — what to do next]
```

### Output format
```
## Framework Map — [Framework name]

[Framework section label]: [mapped content from positioning statement]
[Framework section label]: [mapped content]
[...]

Gaps in the map (missing content before drafting):
- [gap 1 — e.g., "Need a concrete outcome metric for the DESIRE bullets"]
```

## Step 14 — First Draft

### Agent actions
Write the full first draft using the approved framework map.

Rules during drafting:
- Every claim must trace to the approved proof inventory (Phase 2 Step 8)
- Every vague phrase must be replaced with a specific — apply the specificity rule in real-time
- Banned words: none of the banned list from brand-voice.md
- Persona lens: read each sentence as the locked persona — does it resonate, or does it activate their defensiveness?
- Length: obey the length guidelines from brand-voice.md formatting conventions

For assets with multiple sections (landing page, case study, email sequence):
- Write one section at a time
- Present it, get approval, then write the next
- Do not write the full asset in one pass without section-by-section gates

### Output format
Present the draft as clean, published-ready copy. No `[brackets]` left unfilled.
No hedging commentary ("this is just a rough draft...") — show the work.

After the draft:
```
<copy_rationale>
Persona: [name]
Framework: [name]
Primary lever: [the single psychological principle this copy is pulling — e.g., "loss aversion: risk of losing client to competitor who shows up with data"]
Deliberately excluded: [what was NOT said and why — e.g., "did not mention NBC engine — it's 📋 Planned"]
Proof used: [list the specific data points used]
</copy_rationale>
```

## Step 15 — Headline and CTA Variants

### Agent actions
Generate 3 variants for:
1. **The headline** (the single most read element — always worth testing)
2. **The CTA** (the element that drives conversion)

For each variant, state which psychological lever it applies:

## Step 16 — Persona Resonance Test

### Agent actions
Read the full approved draft (with selected headline + CTA) from the perspective
of the locked persona. Apply these five tests:

**Test 1 — First sentence test**
"Does the first sentence speak to the persona's primary frustration or ambition,
or does it talk about SAT?"
(Copy that leads with the product fails this test. Copy that leads with the reader's
world passes it.)

**Test 2 — Archibabble test**
"Is there any phrase that sounds architectural or technical but means nothing specific?"
Flag and fix each instance.

**Test 3 — Specificity test**
"Can a skeptical architect read any claim and immediately ask 'prove it'?"
Flag and fix any vague claims remaining.

**Test 4 — Design autonomy test** (applies when persona = Head of Design)
"Does any phrase imply SAT designs, optimizes, or replaces architectural judgment?"
Flag and fix. Rephrase to: SAT measures / SAT shows / SAT provides data.

**Test 5 — Trust architecture test** (applies when persona = BIM Head)
"Does every technical claim name its data source or mechanism?"
Flag any claim that relies on implicit trust rather than explicit citation.

### Output format
```
## Persona Resonance Test Results

Test 1 (First sentence): [PASS / FAIL — fix if fail]
Test 2 (Archibabble): [PASS / FAIL — flag each instance]
Test 3 (Specificity): [PASS / FAIL — flag each instance]
Test 4 (Design autonomy): [PASS / N/A / FAIL — fix if fail]
Test 5 (Trust architecture): [PASS / N/A / FAIL — fix if fail]

Revised draft (if any tests failed):
[updated copy]
```

## Step 17 — India Market Alignment

### Agent actions
Apply India-specific copy checks before declaring the draft ready for review.

**Check 1 — Regulatory vocabulary**
Does the copy reference Indian standards? If so, verify:
- NBC 2016: is the specific requirement named (not just "NBC-compliant")?
- ECBC 2017: is the specific benchmark named (e.g., energy intensity W/m²)?
- BBMP/BDA: is this Bengaluru-specific copy? Is that intentional?
- Auto-DCR: verify claim with Architecture SME if referenced.

**Check 2 — Vastu bridge (if flagged in Phase 3)**
If deploy = yes, verify:
- The sun/wind analysis output referenced supports a specific Vastu spatial principle
- The framing is empirical ("SAT's data confirms east-facing orientation provides...") not mystical
- The residential developer context is present (who they're selling to)

**Check 3 — Language calibration for Indian B2B buyers**
Indian Tier-1 firms are high-trust, offline-referral-network buyers. Cold digital copy
must acknowledge this reality or it reads as out-of-touch.

Check for these signals:
- Does the copy imply this is a commodity purchase? (bad) → reframe as a relationship / partnership
- Does the copy use US/Western market examples? → replace with Indian firms, Indian cities, Indian regulations
- Does the copy use currency without Indian context? → ensure pricing is ₹-denominated or has India context

**Check 4 — Pilot cohort framing (if relevant)**
SAT is in pilot phase with a limited Tier-1 cohort. If the copy is for outreach:
- "Pilot cohort" framing creates legitimate scarcity without manufacturing urgency
- Verify the pilot cohort detail is accurate (ask human: "Is the 3-firm cohort still current?")

### Output format
```
## India Market Alignment Check

Regulatory vocabulary: [PASS / issues listed]
Vastu bridge: [deployed correctly / skipped / fix needed]
Language calibration: [PASS / issues listed]
Pilot framing: [accurate / needs update / not applicable]

Final draft: [updated copy if any changes made]
```
